The agentic workflow

Five deterministic stages. Each has stage-tagged error handling and cost attribution — a failure surfaces the exact stage that died (script / tts / render / upload / db), never an opaque retry. Model choices are swappable behind an abstraction layer (see Model strategy).

Arabic voice — the decision that defines brand feel

For a Saudi developer this is the whole ballgame: dialect, diacritics, and how project & place names ("Diriyah", "Qiddiya", "Roshn") are pronounced. We don't hand-wave it.

06 / Voice
Option A

Modern Standard Arabic (الفصحى)

Formal Fus'ha for corporate, legal & government-facing copy and pan-Arab reach. Polished and universally understood; available immediately from premium neural TTS.

Option B

Najdi (نجدي)

Central / Riyadh dialect — the default national voice for premium KSA brand campaigns; tests as the most widely-liked dialect for pan-Kingdom consumer copy.

Option C

Hijazi (حجازي) or a Saudi clone

Western / Jeddah dialect — many campaigns run a Najdi + Hijazi pair. Or clone your spokesperson's Saudi-accent voice from ~30s of clean samples.

Week-1 commitment: we demo MSA, Najdi and Hijazi on your script — five of your real project names + an SAR price + a handover date — with a native Saudi-Arabic copy lead on the call, before anything is locked.

Compliance, data & residency

Unreleased pricing, handover dates and renders are sensitive. Two non-negotiables for a UAE developer — regulatory copy and data jurisdiction — handled explicitly.

07 / Trust

Regulatory rule set (enforced, not advisory)

  • Every ad carries its Real Estate Advertisement License number (ترخيص إعلان عقاري) — issued per-ad via REGA's Fal platform & publicly verifiable; distinct from the broker's Fal license
  • Off-plan (البيع على الخارطة) gated to Wafi-registered projects with escrow disclosure; pre-license marketing capped at 5%
  • Price / ROI / guaranteed-return claims gated — REGA bans misleading copy & bait pricing (fines to SAR 200k); generate only when substantiated
  • Mandatory disclosures rendered: advertiser name & capacity, location, valid contact, non-misleading price
  • Named licensed-broker reviewer in the loop before any public asset

Data & residency posture

  • Aligned to Saudi PDPL (enforced by SDAIA, in force Sep 2024) — a cross-border-transfer-restriction regime; documented TRA for sensitive transfers
  • Follows NDMO data-classification & CST Cloud Computing Regulatory Framework for hosting
  • In-Kingdom processing on live regions — Google Cloud Dammam (GA) or Oracle Cloud Riyadh/Jeddah; runs in your tenancy & accounts
  • Sovereign-AI path available: ALLaM (SDAIA/NCAI) & HUMAIN for in-Kingdom inference where required
  • Per-vendor zero-retention / no-training flags; full audit trail of inputs, model, cost & reviewer

Portability

Swap without re-build

One gateway abstracts every model call — script, voice and video. Attribution, zero-retention and fallback configured once. Switching Veo for Runway, or GPT-4o for Claude, is a config change, not an engineering project.

Microsoft-native control plane, multicloud model plane

The architecture splits into two planes. A stable control & integration plane lives entirely on Microsoft 365 / Azure / Entra — where your teams already work, so identity, approvals and assets never leave your existing surfaces. Behind it sits a swappable model plane spread across clouds, hidden behind one model-agnostic gateway. Multicloud exists only to satisfy two facts you can't change — frontier video (Veo) is Google-only, and an in-Kingdom Azure region isn't live until Q4 2026 — and is invisible to the user.

09 / Deployment
Where you already work

M365-native control plane

Operators & approvers sign in with their existing Microsoft Entra ID — no separate login. Human sign-off happens in Teams via Power Automate approval cards; approved assets land in SharePoint as the in-tenant asset library, run state logged to Dataverse for audited, resumable runs.

Models

Azure OpenAI model plane

EN/AR copy & reasoning on Azure AI Foundry (today's Azure OpenAI) — GPT-4o / GPT-4.1 (GA), GPT-5.x at the frontier. OpenAI's Sora 2 video is on Azure AI Foundry in public preview — a Microsoft-native generative-video option alongside Veo.

Routing

Multicloud by design

Every model is reached through one cloud-agnostic gateway with declarative routing. Nothing is load-bearing: Veo on Google Vertex AI, voice on Azure AI Speech or ElevenLabs, sovereign Arabic on HUMAIN's ALLaM — swappable by config, not re-architecture.

Residency

Sovereign & in-Kingdom

Residency-sensitive Arabic & personal-data workloads route to in-Kingdom inference — Vertex AI in Google's Dammam region or HUMAIN's ALLaM. A PDPL scrub/redaction gate ensures only de-identified creative content egresses cross-border.

Honest bridge. Azure has no in-Kingdom region yet — Microsoft confirms Saudi Arabia East goes live in Q4 2026 — so until then, residency-sensitive inference runs in-Kingdom on Google Cloud Dammam (live today, contracted via CNTXT) or HUMAIN ALLaM, then flips to a native Azure endpoint at GA with no application rewrite. Veo itself isn't hosted in Dammam, so frontier video stays a controlled, consented cross-border path either way.
